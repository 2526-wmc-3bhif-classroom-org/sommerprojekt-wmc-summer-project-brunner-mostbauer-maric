# Agent Guidelines: Driving Planer Full-Stack Project

This document provides essential instructions for autonomous agents working on this project. It covers both the backend (Node.js/Express) and frontend (Vue 3/Vite) modules.

## 1. Project Overview
A management system for driving schools.
- **Backend:** Node.js, Express, TypeScript, SQLite (better-sqlite3), Swagger.
- **Frontend:** Vue 3 (Composition API), TypeScript, Vite, PrimeVue, Tailwind CSS, Pinia.
- **Infrastructure:** Docker/Docker Compose for dev (HMR) and prod.

---

## 2. General Infrastructure & Docker
- **Start Dev:** `./start.sh --dev` (maps volumes for Hot Module Replacement).
- **Start Prod:** `./start.sh` (builds optimized images).
- **Stop:** `./stop.sh [--dev]`.
- **Ports:** Backend: 3000, Frontend Dev: 5173, Frontend Prod: 8080.
- **Environment:** Use `.env` files for configuration. NEVER commit secrets or `.env` files.

---

## 3. Backend (Node.js/Express)
**Location:** `/backend`

### Commands
- **Install:** `npm install`
- **Start:** `npm start` (uses `tsx` to run `./src/app.ts`)
- **Dev (with watch):** `npm run dev` (uses `tsx watch` for auto-reloading)
- **Run all tests:** `npm test`
- **Run single test:** `NODE_ENV=test node --experimental-vm-modules node_modules/jest/bin/jest.js path/to/file.test.ts`
- **API Docs:** Accessible at `http://localhost:3000/api-docs` when running.

### Code Style & Guidelines
- **Project Structure:** Domain-driven folders (e.g., `src/users/`).
  - `*_router.ts`: Express routes + `@swagger` JSDoc annotations.
  - `*_service.ts`: Business logic & transaction management.
  - `*_repository.ts`: SQL queries (using `better-sqlite3`).
- **Imports:** Always use `.js` extensions in imports (ESM requirement).
  - Example: `import { UserRole } from "../models/types.js";`
- **Database Columns:** Use `PascalCase` (matching SQL schema, e.g., `UserId`).
- **Types:** Shared interfaces (except `User`) are in `src/models/types.ts`.
- **Unit of Work:** Use the `Unit` class for all DB operations.
  - Read-only: `Unit.createReadonly()`.
  - Transactions: `new Unit(false)`.
  - **CRITICAL:** Always call `unit.complete(success)` in a `finally` block for transactions.
- **Error Handling:** Use `http-status-codes`. Wrap router handlers in try-catch.
  - Response format: `{ error: { message: string } }` or `{ data: T }`.
- **Enums:** Use the `UserRole` enum for roles (ADMIN, USER, SCHOOL).
- **School Accounts:** School accounts are identified by `Role === UserRole.SCHOOL`. No separate `IsSchool` boolean exists in the database.

## 4. Frontend (Vue 3/Vite)
**Location:** `/frontend/driving_planer`

### Commands
- **Install:** `npm install`
- **Dev Server:** `npm run dev`
- **Build:** `npm run build`
- **Lint/Format:** `npm run lint`, `npm run format` (Prettier).
- **Type Check:** `npm run type-check`.
- **Unit Tests:** `npm run test:unit`.
- **Run single test:** `npx vitest path/to/test.ts`

### Code Style & Guidelines
- **Framework:** Vue 3 Composition API with `<script setup lang="ts">`.
- **Styling:** Tailwind CSS utility classes + PrimeVue components.
- **State Management:** Pinia stores.
- **API Communication:** Use `import.meta.env.VITE_API_URL`.
- **Naming:** `PascalCase` for Components, `camelCase` for variables/methods.
- **Quality:** Prioritize type safety. Avoid `any`. Run `npm run format` before finishing.

---

## 5. Global Agent Safety & Best Practices
- **Git:** NEVER run `git` commands. Provide them for the user if requested.
- **File Deletion:** Always ask for explicit confirmation before deleting files/directories.
- **Testing Strategy:** 
  - Backend: Jest + Supertest. Tests co-located with code (e.g., `auth_router.test.ts`).
  - Frontend: Vitest + Vue Test Utils.
- **Generative AI Attribution:** 
  - Add `// These tests were generated by Gemini` at the top of new test files.
  - Add `// This swagger comment was generated by gemini` above new `@swagger` blocks.
- **Transactions:** Ensure `unit.complete(commit)` is called correctly to prevent "transaction has been opened" errors.
- **JSON Safety:** The backend includes middleware to handle malformed JSON; ensure your implementations don't bypass this.
